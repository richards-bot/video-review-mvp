# Project Context

## Identity

- **Name:** Secure Video Review MVP
- **Mission:** Share legally-sensitive video with reviewers via an encrypted browser link — no keys ever leave the sender's device or reach the storage backend.
- **Target Users:** Journalists, lawyers, and investigators who need to share sensitive video with external reviewers without trusting cloud infrastructure.

## Status

- **Phase:** Active Dev (MVP complete; v1.1 roadmap open)
- **Codebase:** Greenfield (purpose-built, no legacy migration)
- **Test Coverage:** Comprehensive — Go packages 80%+ unit coverage; crypto 100%; viewer JS 47 Vitest tests; all checks green

## Technical Context

- **Languages:** Go 1.22 (CLI), vanilla JavaScript/ES2022 (viewer)
- **Frameworks:** None — stdlib Go, vanilla JS with Web Crypto API and MSE
- **Data Stores:** AWS S3 (encrypted blobs only — no plaintext ever stored)
- **External Services:** AWS S3, AWS KMS (optional envelope key), CloudFront (optional CDN)
- **Build toolchain:** Bundled Go at `.tools/go/bin/go`; viewer has no build step

## Constraints

- AES-256-GCM encryption is non-negotiable — S3 must never see plaintext
- No server-side components in the viewer path — static HTML/JS only
- Decryption key travels only in the URL `#fragment` — never sent to any server
- No WASM or build step in the browser viewer — pure ES modules
- ffmpeg/ffprobe required for the uploader (transcodes to fMP4) but is NOT present in this dev environment — integration tests requiring it must be skipped

## Domain Terminology

| Term | Definition |
|------|------------|
| CEK | Content Encryption Key — 32 random bytes generated per upload, shared only in the URL fragment |
| asset_id | 32-char hex string that is the S3 prefix for all chunks and the manifest |
| manifest | Encrypted JSON file listing all chunk IDs, MIME type, and total duration |
| chunk | One encrypted fMP4 segment (up to 8 MB plaintext) stored as `chunk_NNNNNN` in S3 |
| fMP4 | Fragmented MP4 — required for Media Source Extensions streaming in the browser |
| share URL | `https://viewer.example/index.html#<asset_id>.<base64url(CEK)>` — fragment never reaches S3 |
| MSE | Media Source Extensions — browser API for streaming progressive media |
| SRI | Subresource Integrity — `integrity=` attribute on `<script>` tags in the viewer |
| presigned URL | Temporary AWS S3 URL generated by the viewer to fetch chunks without exposing credentials |
