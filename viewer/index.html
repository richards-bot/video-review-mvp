About security
Security properties of this viewer

Content is encrypted with AES-256-GCM before upload to S3. The decryption key is contained in the URL fragment and is never sent to any server. S3 stores only ciphertext and has no access to the key.

Limitations: This system does not prevent a reviewer from screen-recording the content, does not watermark the video, cannot revoke access once a URL is shared (presigned URLs expire after 48 h by default), and does not verify reviewer identity. Anyone with the URL can view the content — the URL is the credential.

For more information, see the project README.
